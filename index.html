Abstract

This document serves as the official registry of powerful features of the web platform, providing a centralized place to find these features.

Status of This Document

This section describes the status of this document at the time of its publication. A list of current W3C publications and the latest revision of this technical report can be found in the W3C technical reports index at https://www.w3.org/TR/.

The list of features in this registry is not exhaustive. It is expected that new features will be added to this document as they are implemented across multiple user agents.

Please subscribe to the Powerful Features Registry Repository on GitHub to be notified of changes.

1. Registry table of powerful features

Powerful features are identified by a unique string token, which is used for integration with the Permissions and Permissions Policy specifications.

Note: Permissions and Permissions Policy

Not all permission policy tokens are powerful features (e.g., "sync-xhr"), but every powerful feature is a policy-controlled feature. Please refer to the Permissions specification for guidance on how to specify a powerful feature.

Being listed in the table below does not imply that a user agent supports a powerful feature.

Powerful features of the web platform
Unique token that identifies the powerful feature Specification that defines the powerful feature
"accelerometer" Accelerometer
"ambient-light-sensor" Ambient Light Sensor
"background-fetch" Background Fetch
"background-sync" Web Background Synchronization
"bluetooth" Web Bluetooth
"camera" Media Capture and Streams
"display-capture" Screen Capture
"geolocation" Geolocation API
"gyroscope" Gyroscope
"magnetometer" Magnetometer
"microphone" Media Capture and Streams
"midi" Web MIDI API
"nfc" Web NFC API
"notifications" Notifications API Standard
"persistent-storage" Storage Standard
"push" Push API
"screen-wake-lock" Screen Wake Lock API
"speaker-selection" Audio Output Devices API
"xr-spatial-tracking" WebXR Device API
